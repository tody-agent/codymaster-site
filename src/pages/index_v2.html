⚡ Tiến hoá

Mỗi Bug bạn fix
khiến nó mạnh hơn.

Tool thường hỏng và nằm im. Symbiont của CodyMaster biến mọi thất bại thành nhiên liệu. Bug trở thành kháng thể. Crash trở thành sự bền bỉ. Nỗi đau trở thành sức mạnh.

🐛 → 🛡️
Bug → Kháng thể
Bug xuất hiện → Root cause analysis (không đoán, điều tra) → Fix với TDD → Learning được capture vào learnings.json → Lần sau gặp pattern tương tự → tự phòng ngừa. Bug đó không bao giờ xuất hiện lần thứ 3.
💥 → 🧠
Crash → Trí tuệ
Deploy fail → Rollback tự động → Post-mortem captured → Architecture decision documented → Kiến thức được promote qua Ebbinghaus reinforcement → Next deploy an toàn hơn. Mỗi crash là một bài học vĩnh viễn.
😵 → 🔬
Hỗn loạn → Hệ thống
Code spaghetti → cm-design-system scan toàn bộ → Extract patterns → Build design system → Enforce consistency. Từ hỗn loạn sinh ra trật tự. Từ legacy code sinh ra kiến trúc sạch.

Vấn đề gặp 2 lần thì không được phép gặp lần thứ 3. Đó là triết lý của kẻ lười biếng thông minh — và cũng là cách CodyMaster tự tiến hoá.

🛡️ Hệ Miễn dịch

8 Cổng Miễn dịch.
Không gì lọt qua.

Giống hệ miễn dịch cơ thể người — da, chất nhầy, bạch cầu, kháng thể — CodyMaster bảo vệ codebase bằng 8 cổng tuần tự. Mỗi cổng là một loại tế bào miễn dịch khác nhau.

G1
🔬 Static Analysis
Hàng rào Da — Lớp bảo vệ đầu tiên
Bắt lỗi syntax, lint, type errors. Không qua được lớp này thì đừng mơ vào sâu hơn.
G2
🧪 Test Suite
Bạch cầu — TDD enforcement
Mọi test phải pass. Viết test trước, code sau. RED → GREEN → REFACTOR. Không ngoại lệ.
G3
👁️ Blind Review
Tế bào NK — Đánh giá không thiên vị
Reviewer chỉ thấy diff — không task description, không implementation context. Buộc phải review khách quan thực sự.
G4
😈 Anti-Sycophancy
Tế bào T — Devil's Advocate
Nếu Gate 3 nói "mọi thứ ổn", Gate 4 chủ động săn tìm vấn đề ẩn. Không cho phép tự mãn. Không nịnh hót.
G5
🔒 Security Scan
Kháng thể — Quét lỗ hổng liên tục
Secret detection pre-commit, Snyk Code SAST, vulnerability scanning. Quét liên tục, ngăn ngừa lỗ hổng. Không secret nào lọt qua.
G6
🌐 i18n Check
Tính toàn vẹn màng tế bào
Mọi string đều được extract, không hardcode text. Đảm bảo tính nhất quán đa ngôn ngữ.
G7
🏗️ Build Verification
Kiểm tra cơ quan nội tạng
Production build phải thành công. Bundle size check, dependency audit, dead code elimination.
G8
💚 Health Check
Dấu hiệu sinh tồn — Khám sức khoẻ post-deploy
Smoke test + uptime verification sau deploy. Nếu fail → auto-rollback ngay lập tức. Không bao giờ để hệ thống chết.

🔧 Tự sửa chữa

🐛
Bug Detected — Phát hiện lỗi
Hệ thống phát hiện anomaly qua test suite hoặc runtime monitoring.
🔬
Root Cause Analysis — Điều tra gốc rễ
cm-debugging phân tích có hệ thống. Không đoán, không thử random. Tìm nguyên nhân thật sự.
🧪
Reproduce Test — Tái hiện bằng test
cm-tdd viết test MÀ TÁI HIỆN bug trước. Red test = chứng minh bug tồn tại.
💊
Fix — Chữa trị
Sửa code cho test chuyển từ RED → GREEN. Refactor nếu cần.
🛡️
8-Gate Verify — Kiểm tra toàn bộ
Chạy toàn bộ 8 gate để đảm bảo fix KHÔNG phá vỡ bất cứ thứ gì khác. Phân tích tổng thể.
🧠
Capture Learning — Ghi nhớ vĩnh viễn
cm-continuity lưu pattern lỗi này thành kháng thể. PRE-ACT ATTENTION kiểm tra trước MỌI hành động tương lai: "Đã gặp pattern này chưa?" → Tự phòng ngừa.

Max 3 lần thử mỗi task. Sau 3 lần thất bại → BLOCKED + ESCALATE cho con người. Không lặp vô hạn. Không phá hệ thống. Luôn phân tích tổng thể trước khi hành động.
