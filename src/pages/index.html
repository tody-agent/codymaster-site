Self-Healing AI

An immune system for your logic.

When a skill fails or encounters unknown parameters, the system doesn't just crash. It invokes cm-skill-health to diagnose the failure, and cm-skill-evolution to automatically generate a mutated, fixed version of the skill. Over time, your AI becomes perfectly adapted to your specific codebase.

Workflow Orchestration

Skills chain together. Automatically.

The animated particle streams between nodes represent workflow chains. Each workflow is a pipeline of skills that fire in sequence — from idea to deployed product. No manual coordination needed.

TYPICAL PIPELINE

cm-start → brainstorm → planning → execution → quality-gate → safe-deploy

Skills auto-redirect when complexity is detected: cm-planning detects ambiguity → redirects to cm-brainstorm-idea. The brain self-corrects its workflow path.

QMD Embedding + NotebookLM

Local search meets cloud intelligence.

The outer sphere represents the knowledge layer. qmd (by Shopify’s Tobi) provides BM25 search across massive local docs — feeding the Semantic Memory tier. NotebookLM extends this to the cloud — syncing high-value knowledge across machines, generating podcasts, flashcards, and queryable endpoints.

🔍 qmd: Local BM25 search over docs & code
☁️ NotebookLM: Cross-machine sync
🎙️ Auto-podcast from codebase knowledge
📇 Flashcards for human onboarding

Data flow: Local Brain → brain-sync.sh → Cloud NotebookLM → Podcast / Flashcards / Query API
